# นิดเดียว Badminton Club — Design Context

## Design Context

### Users
- **สมาชิกก๊วน** (public, read-only) — เปิดจากมือถือที่สนามแบด ดูอันดับ/สถิติตัวเอง, เช็คคู่ปรับ ("เทกันจัง"), ดูผลแมตช์ ใช้แบบเร็วๆ ระหว่างพักเบรก
- **Admin** (2-3 คน) — จัดการเช็คอิน, จับคู่ตาม ELO, คิดเงินหลังเล่นจบ ผ่านมือถือ/แท็บเล็ตข้างสนาม ต้องกดง่าย เร็ว ไม่ใช้เวลาอ่านนาน

### Brand Personality
**Esports/Gaming ดิบ เข้ม มันส์** — ดุดัน (aggressive), high-energy, competitive. ไม่ใช่ corporate/SaaS dashboard ทั่วไป
โลโก้: แมวมงกุฎถือไม้แบด สี hot pink บนพื้นดำ

### Aesthetic Direction
- **Dark-only**, high contrast — ไม่มี light mode
- สี: `#0a0a0a` (black) + `#ec4899` (hot pink) เป็นคู่หลัก
- แนว esports HUD/scoreboard (ระดับ Valorant/Riot UI spirit) ไม่ใช่ generic admin panel
- ธีมระดับมือ 5 ขั้นตามเครื่องดื่ม: Milk → Soju → Beer → Highball → Vodka — ควรเป็น visual language จริงจัง ไม่ใช่แค่ badge สีเฉยๆ
- ไม่มี reference เว็บเจาะจง — ออกแบบอิสระ
- **Anti-reference**: ของเดิมที่มีอยู่ตอนนี้ (การ์ดโค้งมนธรรมดา, เงาเบาๆ, spacing เท่ากันหมด) คือสิ่งที่ต้องการหลีกเลี่ยง — รู้สึก generic เกินไป
- Mobile-first เข้มงวด — ส่วนใหญ่เปิดจากมือถือหน้าคอร์ท ไม่ใช่จอใหญ่

### Design Principles
1. Contrast สูง คมชัด ดิบ — ไม่ soft/pastel/timid
2. Typography ต้องมีคาแรกเตอร์ ไม่ใช่ font default ของเบราว์เซอร์ และต้องรองรับภาษาไทยสวย
3. หลีกเลี่ยง generic AI design tells: card grid เหมือนกันหมด, border-left accent stripe, gradient text, glassmorphism พร่ำเพรื่อ
4. Motion ใช้ตอน state สำคัญเปลี่ยน (เช็คอิน, ชนะแมตช์, ELO อัพเดท) ให้รู้สึก "มันส์" ไม่ใช่ animation ประดับทั่วไป
5. Admin screens ต้องกดง่าย เร็ว ด้วยนิ้วโป้งข้างสนาม — ปุ่มใหญ่พอ, ลำดับขั้นตอนชัด
